üîç Bagaimana XSS Bekerja?

Reflected XSS

‚ùå Vulnerable Code: <?php echo "Search results for: " . $_GET['q']; ?> ‚ùå URL yang dihasilkan: https://example.com/search.php?q=<script>alert('XSS')</script>

Stored XSS

‚ùå Vulnerable Code: <?php $comment = $_POST['comment']; // Simpan ke database tanpa sanitasi $query = "INSERT INTO comments (content) VALUES ('$comment')"; // Tampilkan komentar echo "<div>" . $row['content'] . "</div>"; ?>

DOM-based XSS

‚ùå Vulnerable JavaScript: var name = location.hash.substring(1); document.getElementById('welcome').innerHTML = "Hello " + name;

üõ°Ô∏è Cara Pencegahan XSS

1. Input Validation & Output Encoding

‚ùå Vulnerable: echo $_GET['input']; ‚úÖ Secure: echo htmlspecialchars($_GET['input'], ENT_QUOTES, 'UTF-8'); // JavaScript ‚ùå Vulnerable: element.innerHTML = userInput; ‚úÖ Secure: element.textContent = userInput;

2. Content Security Policy (CSP)

‚úÖ CSP Header: Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data:;

3. HTTP-Only Cookies

‚úÖ Secure Cookie: setcookie("session", $value, [ 'httponly' => true, 'secure' => true, 'samesite' => 'Strict' ]);

4. Framework Protection

‚úÖ React (Auto-escaping): const message = "<script>alert('XSS')</script>"; return <div>{message}</div>; // Otomatis di-escape ‚úÖ Laravel Blade: {% raw %} {{ $userInput }} // Otomatis di-escape {!! $trustedInput !!} // Raw output (hati-hati!) {% endraw %}

‚ö° Dampak Serangan XSS

1. Session Hijacking

Penyerang dapat mencuri cookie session dan mengambil alih akun pengguna.

Document.cookie: sessionid=abc123; userid=12345; role=admin

2. Phishing

Membuat form palsu untuk mencuri kredensial pengguna.

3. Malware Distribution

Mengarahkan pengguna ke situs berbahaya atau mengunduh malware.

4. Defacement

Mengubah tampilan website untuk merusak reputasi.
